# DESARROLLO

-Desarrollar con patrones y estructuras con Clean Code.
-Cumplir principios SOLID.
-Cumplir el principio de responsabilidad única.
-Desarrolla siempre funcionalidades robustas y escalables, pensando principalmente en la seguridad.
-Desarrollar con patrones actuales de desarrollo de Frontend.

# DOCUMENTACION

-Usa la carpeta /docs para mirar/estudiar/almacenar toda la documentación técnica sobre la plataforma.
-Se debe mantener un orden estructural y organizado de carpetas para esta documentación.
-Se debe generar documentación técnica de cada funcionalidad, esta documentación debe servir como manuales explicativos y de integración para usuarios avanzados e informáticos.
